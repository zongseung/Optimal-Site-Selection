# Phase 3 기획서 — 토지피복 발화신호 + 조건부 풍하노출

**작성** 2026-06-19 · **전제** Phase 1·2 완성(물리 MoE + 발화점 앵커, 공간CV recall@top5% 0.10). 본 문서는 두 가지 개선을 추가한다.

---

## 0. 지금까지 결과 — 쉽게

- 전주 1,387,831개 각각에 **위험(1)/안전(0)**을 매겨 제출. 지금은 가장 위험한 **2%(27,757개)**를 위험으로 표시.
- "잘 맞히나?"를 과거 발화점으로 채점하면: **가장 위험한 5% 전주 안에 과거 화재의 약 10%가 들어옴**(아무렇게나 찍으면 5% → 우리는 그 2배). 상위 10%면 화재의 17%(랜덤 10% → 1.7배).
- 즉 **방향은 맞지만 아직 세지 않다.** 이유: 채점 기준인 과거 발화점이 *사람이 산에서 낸 불* 위주라 "전주 설비 위험"과 완전히 겹치지 않음(우리가 가진 유일한 객관 잣대의 한계).
- 풍하 확산 시뮬은 **물리적으론 정확**(2019 고성에서 불이 동쪽으로 번진 실제 흉터와 일치)하지만, 그걸 위험 순위에 섞으니 점수가 오히려 떨어져서 지금은 **참고용 컬럼**으로만 둠.

→ Phase 3에서 이 둘을 보강한다.

## 1. 개선 ① — 토지피복을 발화항(I)에 반영 (사용자 지정)

**근거**: 발화의 91%가 산림(전주 중 산림은 18%뿐) → 토지유형이 발화 성향을 강하게 가른다.

**무엇을 한다**
- 전주 `lc_group`(농업/초지/산림/시가화/나지/습지/수역)에 **토지피복 발화계수 `lc_ignition`**(0~1)을 부여. 값은 위 "발화 성향 배율"에서 유도한 **도메인 사전치**(발화점 라벨을 학습에 직접 쓰지 않음 — EDA 사실에 근거한 prior):
  - 산림 1.00 · 초지 0.25 · 나지 0.15 · 농업 0.10 · 시가화 0.08 · 습지 0.02 · 수역 0.00
- 이 `lc_ignition`을 **발화항 I의 새 성분 `landcover`**로 추가(기존 forest/road/powerline/fwi/yanggan/fuel + landcover = 7성분). `experts.py`.
- `EXPERT_WEIGHTS`에 체제별 `landcover` 가중 추가 후 **가중치 재튜닝**(기존 공간CV recall 목적·동일 하네스)으로 7성분 균형 재조정.

**주의(정직)**: 산림 토지피복은 `dist_to_forest`(산림까지 거리)와 의미가 겹친다(전주가 *숲 안*인지 vs *숲 근처*인지). 둘은 보완적이라 같이 두고, 중복은 튜닝이 흡수. 재튜닝 후 어떤 성분이 커지는지 해석 보고.

**기대/검증**: 공간CV recall@top-k 상승 기대(산림 전주 정렬↑). before/after 비교, 낙관편향(무작위-공간 격차≈0) 유지 확인.

## 2. 개선 ② — 조건부 풍하노출 (사용자 지정 "2번 반영")

**문제 복기**: 풍하노출(exposure)을 *전 전주에 일괄* 위험순위에 섞으니 균질해져 recall이 떨어졌다(Phase 2).

**아이디어**: 노출(=바람타고 번짐)은 **바람이 지배하는 곳에서만** 의미가 크다 → **영동(양간지풍) 고위험 맥락에만 조건부로** 반영.

**무엇을 한다** — 전주별 조건부 블렌드 강도 `α_p`:
```
α_p = α_base × (영동 게이트 가중)_p × (양간 강도 정규화)_p        # 0~α_base
S_final(p) = (1 − α_p)·S_static(p)  +  α_p·Pexposure_norm(p)
```
- 영동·고양간 전주만 노출이 S에 실효적으로 들어가고, 영서·산간은 사실상 정적 S 유지(전역 희석 방지).
- (대안 경로) 일별 엔진에서는 **양간지풍 고위험일(고FWI·고풍속)에만** 노출 시뮬을 적용하는 조건부도 가능 — 시즌집계에 반영.

**검증·결정 규칙(정직)**:
- 공간CV recall을 **전체 + 영동 부분집합**으로 각각 측정. 2019 고성 풍하 sanity 재확인.
- **조건부 노출이 CV recall을 올리면 채택**(α_base를 CV로 선택). **중립/악화면 decision에는 미반영**하고, 영동 분석·제출 컬럼(p_exposure)·활용방안(조기경보 풍하 경고)으로만 유지. ← 점수를 해치지 않는다는 원칙 고수.

## 3. 작업·검증 체크리스트
1. `io.py`에 lc_group 로드(이미 경로 있음) → `experts.py`에 `landcover` 성분 + `lc_ignition` 사전치(config 상수화).
2. `config.EXPERT_WEIGHTS`에 `landcover` 추가 → `scripts/tune_weights.py` 재실행(7성분), provenance 갱신.
3. `hazard.py`/`exposure_engine.py`에 조건부 α_p 블렌드 구현(영동게이트×양간).
4. 공간블록CV recall@top-{1,2,5,10}% **before/after(① 토지피복, ② 조건부노출 각각·합산)**, 영동 부분집합, 낙관편향, 고성 sanity.
5. 제출물 재생성(1,387,831행·0/1·양성비율 보고), 테스트 통과 유지, RAYON_NUM_THREADS=50 유지.
